VibeFeedback PoC

Teile eine Live-URL deines Vibecoding-Projekts. Feedback-Geber:innen klicken direkt auf die Elemente, die sie kommentieren wollen. Am Ende bekommst du alles gesammelt als Markdown-Prompt zurück.

Sharing-Link erzeugen

1
Deploye dein Projekt (z. B. GitHub Pages, Vercel, Netlify) und kopiere die URL zur HTML-Seite.
2
Füge sie unten ein — VibeFeedback verpackt sie in einen teilbaren Feedback-Link.
3
Schicke den Link an deine Tester. Sammle das Feedback später als Markdown und füttere damit deinen nächsten Prompt.
Öffnet den Link im Owner-Modus. Feedback wird im Browser gespeichert (localStorage).

Wie das PoC funktioniert

  • Die geteilte HTML wird per fetch() geladen und in ein Iframe eingebettet — das erlaubt Klick-Zuordnung zu konkreten DOM-Elementen.
  • Klicks im Iframe öffnen ein Kommentarfeld. Der CSS-Selector + HTML-Auszug des Elements wird als Referenz gespeichert.
  • Kommentare liegen im localStorage pro Browser. Kein Server, kein Login.
  • Kommentierer exportiert am Ende ein Markdown und schickt es dir (Mail/Chat) — direkt Prompt-tauglich.
  • CORS-Hinweis: Das Ziel muss GET-Requests via CORS erlauben (GitHub Pages tut das). Sonst Fallback: HTML-Datei paste-in.
